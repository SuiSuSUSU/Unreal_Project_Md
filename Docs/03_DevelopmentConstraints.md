# 개발 제약 조건

이 문서는 개발자의 현실적인 조건을 정리한다. 게임 방향은 이 제약 안에서 결정해야 한다.

## 강점

- 개발자는 3D 모델러이므로 모델링, 실루엣, 공간감, 오브젝트 디자인을 강점으로 삼을 수 있다.
- Unreal Engine 5.7을 사용하므로 3D 렌더링, 조명, 머티리얼, 에디터 워크플로를 활용할 수 있다.
- Codex를 통해 반복적인 코드 작성과 구조 정리를 보조받을 수 있다.
- 1인 개발이므로 의사결정이 빠르다.

## 약점

- 코딩을 직접 깊게 다루기 어렵다.
- 복잡한 시스템을 장기간 유지보수하기 어렵다.
- 블루프린트와 C++ 연결에서 사람의 확인 작업이 필요할 수 있다.
- 테스트, 밸런싱, UI, 사운드, 배포까지 혼자 판단해야 한다.

## 피해야 할 개발 방향

- 대규모 오픈월드
- 복잡한 멀티플레이
- 많은 캐릭터와 퀘스트가 필요한 RPG
- 복잡한 경제, 제작, 시뮬레이션 시스템
- 많은 예외 규칙이 필요한 전투 시스템
- Codex가 한 번에 이해하기 어려운 거대한 기능 단위

## 적합한 개발 방향

- 핵심 루프가 단순한 게임
- 작은 기능을 하나씩 추가해도 플레이가 발전하는 구조
- 모델링 결과물이 바로 차별점이 되는 구조
- 반복 가능한 스테이지, 웨이브, 짧은 세션 중심 구조
- 코드보다 에셋과 연출의 비중이 높은 구조

## Codex에게 맡기기 좋은 작업

- C++ 클래스 생성과 기본 로직 구현
- 스폰, 이동, 공격, 체력, 경험치 같은 독립 기능
- 데이터 테이블이나 설정값 기반 구조 제안
- 기존 코드 오류 수정
- 빌드 오류 분석
- Markdown 기획 문서 정리
- 구현 후 수정 파일과 테스트 절차 요약

## 사람이 직접 판단해야 하는 작업

- 최종 게임 방향 결정
- 재미와 조작감 판단
- 모델링 콘셉트와 비주얼 방향
- 에셋 퀄리티 기준
- 난이도와 속도감 체감
- 출시 목표와 작업 범위 조정
- 블루프린트/에디터에서 실제 배치와 플레이 확인

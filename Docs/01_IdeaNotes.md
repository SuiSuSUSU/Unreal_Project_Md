# 아이디어 노트

이 문서는 대화 중 떠오른 게임 아이디어를 자유롭게 기록하는 공간이다. 확정안과 후보/보류 아이디어를 구분해서 관리한다.

## 작성 규칙

- 확정된 내용처럼 쓰지 않는다.
- 아이디어는 짧게 기록하고, 나중에 비교 문서에서 검토한다.
- 1인 개발 현실성과 Codex 구현 가능성을 함께 본다.
- 보류된 아이디어도 삭제하지 않고 이유를 남긴다.

## 떠오른 게임 아이디어

| 아이디어 | 장점 | 단점 | 현재 상태 |
| --- | --- | --- | --- |
| 스테이지 클리어형 로그라이트 | 콘텐츠를 단계별로 확장하기 좋음 | 방, 보상, 진행 구조 설계가 필요 | 우선 방향 |
| 방 클리어형 카타나 액션 | 현재 콤보/회피/적 구조를 유지하며 방 단위 전투와 보상을 붙일 수 있음 | 방 관리, 문, 보상, 다음 구역 이동 구조가 필요함 | 우선 방향 |
| 뱀파이어 서바이벌류 3D 생존 액션 | 반복 구조가 명확하고 작은 기능 단위로 구현 가능 | 현재 목표인 방 클리어형 진행과는 루프가 다름 | 보류 |
| 공장/기계 테마 액션 | 3D 모델러의 하드서피스 강점을 살리기 좋음 | 시스템이 커지면 구현 부담 증가 | 후보 |
| 디펜스형 게임 | 규칙이 명확하고 UI/웨이브 구조를 만들기 쉬움 | 현재 조작/콤보 액션 방향과 거리가 있음 | 보류 |
| 짧은 호러/탐색 게임 | 작은 범위에서도 분위기와 모델링 강점을 살릴 수 있음 | 연출, 사운드, 레벨 디자인 의존도가 높음 | 후보 |
| VR 리듬/레이싱 게임 | 시각적 차별화 가능 | VR 입력, 최적화, 테스트 장비 부담이 큼 | 추가 검토 |

## 현재 우선 방향

- 작은 방/구역을 하나씩 클리어하는 스테이지 돌파형 탑다운 액션 MVP.
- 초기에는 같은 맵 안에 고정된 방 2~3개를 배치한다.
- 방 입장, 문 닫힘, 정해진 적 스폰, 적 전멸, 방 클리어, 보상 획득, 다음 방 이동을 핵심 루프로 본다.
- 현재 구현된 WASD 이동, 좌클릭 1/2/3타 콤보, Space 회피, `HealthComponent`, `EnemyBase`, `EnemyStatsDataAsset`, Little Demon 적 구조는 유지한다.

## 1인 개발 기준 리스크

- 방 관리, 문, 보상, 다음 구역 이동까지 붙으면 시스템 수가 늘어난다.
- 방마다 적 배치와 보상 템포를 직접 조정해야 한다.
- 랜덤 던전, 상점, 열쇠, 복잡한 아이템 시너지를 초기에 넣으면 범위가 커진다.
- UI, 저장, 밸런싱, QA도 혼자 처리해야 한다.

## Codex 의존 개발 기준 리스크

- 큰 기능을 한 번에 맡기면 의도와 다르게 구현될 수 있다.
- 방 단위 전투는 `RoomCombatActor`, `RoomSpawnPoint`, `RoomDoorActor`, `RewardActor`처럼 작은 단위로 나눠 구현해야 한다.
- 블루프린트 에셋 연결은 사람이 직접 확인해야 할 수 있다.
- 기획 문서가 모호하면 코드 구조도 흔들릴 수 있다.

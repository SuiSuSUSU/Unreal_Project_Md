## 2026-06-24 Update: Run Reward Guardrails

- `PlayerStatsComponent` now ignores non-finite run reward bonus values before changing runtime modifiers.
- This guard applies to max health, basic attack damage, move speed, and dodge cooldown run rewards.
- The guard is a safety fallback only; normal reward cards should still provide valid finite values.
- `FRewardCardOption.RewardId` identifies a reward card, but stat changes still go through `PlayerStatsComponent`.
- Reward cards must not modify `PlayerStatsDataAsset` directly.
- Future reward data assets or reward tables should describe reward intent, while `PlayerStatsComponent` remains the runtime stat application point.

## 2026-06-23 Update: Runtime Run Reward Modifiers

- `PlayerStatsComponent` now supports temporary run reward modifiers for the current play session.
- Current run modifier targets are max health, basic attack damage, move speed, and dodge cooldown.
- These modifiers are runtime-only and must not write back to `PlayerStatsDataAsset`.
- `Max Health +10` reward uses the runtime max-health bonus path and then updates `HealthComponent` max health.
- `Run Attack Damage Up` affects values read by the basic attack subsystem through `PlayerStatsComponent`.
- `Run Move Speed Up` temporarily adjusts the owning character's `CharacterMovement.MaxWalkSpeed`.
- `Run Dodge Cooldown Down` reduces the runtime dodge cooldown returned by `PlayerStatsComponent`.
- Run modifiers reset on player death through `PlayerHealthSubsystem`, and naturally reset on level restart because the component is recreated.

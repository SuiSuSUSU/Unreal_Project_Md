# HealthSystem.md

## 2026-07-01 Update: Enemy White Hit Flash Removed

- `EnemyBase` no longer uses `HealthComponent.OnHealthChanged` to trigger enemy white hit-flash feedback.
- Enemy damage still flows through the shared `HealthComponent` damage and death events.
- Player health UI can still listen to `OnHealthChanged`; only the enemy white flash presentation path was removed.
- Future enemy hit feedback should be designed separately from the core health component.

## 2026-07-02 Update: HitReact Health Boundary

- `HealthComponent` should remain the shared health, damage, heal, and death event component.
- `HealthComponent` should not choose flinch animations, stagger rules, impact VFX, camera shake, or enemy-specific hit reactions.
- A future `HitReact` path may listen to confirmed damage through a receiver such as `EnemyBase`, but the target-side reaction policy should stay outside `HealthComponent`.
- Death handling should remain separate from non-lethal hit reaction. Do not play normal flinch reactions after an enemy is dead.
## 2026-06-24 Update: Reward Health Safety

- Reward card UI can open even if the player `HealthComponent` is temporarily missing at pickup overlap time.
- Health-specific reward effects still validate `HealthComponent` when the card is selected.
- If `HealthComponent` is missing during a health reward effect, the effect is skipped and a warning is logged instead of crashing.

## 2026-06-23 Update: Reward Health Apply Types

- `Heal +20` is now explicitly an `Instant` reward and calls `HealthComponent::Heal`.
- `Max Health +10` is now treated as a `Run` reward through `PlayerStatsComponent` runtime modifiers.
- Runtime max-health rewards update `HealthComponent::SetMaxHealth` after the run modifier is applied, then heal by the bonus amount.
- The player's run reward modifiers are reset when `PlayerHealthSubsystem` handles player death.
- Permanent health upgrades are not implemented yet and should not write save data in the current MVP.

## 2026-06-21 Update: Reward Card Health Effects

- The reward card MVP currently includes health-related test effects.
- `Heal +20` calls `HealthComponent::Heal(20)`.
- `Max Health +10` calls `HealthComponent::SetMaxHealth()` and then heals the added amount.
- These are temporary card effects for testing the reward choice flow.

## 2026-06-21 Update: Previous Room Reward Heal MVP

- The first `RewardActor` test used direct `HealthComponent::Heal(20)` on pickup.
- The current direction has moved that heal into a selectable reward card.
- Direct heal pickup is no longer the current room reward behavior.
